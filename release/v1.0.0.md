# Wiz VSCode Extension v1.0.0 Release Notes

## Summary

**v1.0.0** is the initial release of the Wiz VSCode Extension, providing a comprehensive development environment for Wiz Framework projects with a custom tree explorer, specialized editors, virtual file system, and full project management capabilities.

---

## Core Features

### Smart Project Explorer
- **Three-Level Structure**: Source, Packages (Portal), and Project categories
- **App Type Recognition**: Automatic detection of Page, Component, Layout, and Route apps
- **Virtual Folders**: Display standard folders even when they don't exist yet
- **Auto-Highlighting**: Automatically reveals the active file in the tree view
- **Drag & Drop**: Move files and folders between locations
- **Multi-Select**: Work with multiple items simultaneously

### Specialized Editors
- **App Info Editor**: Webview-based visual editor for `app.json` configuration
- **Route Editor**: Dedicated interface for route-specific settings with flat structure support
- **Portal App Editor**: Namespace-synced editor for portal applications (folder = namespace = ID)
- **Portal Package Editor**: Manage `portal.json` with auto-completed fields
- **View Type Selection**: Choose between HTML and Pug templates

### Virtual File System (`wiz://`)
- Custom URI scheme for opening app files (UI, Controller, API, etc.) as clean editor tabs
- Format: `wiz://{authority}/{base64-path}?label={display-name}`
- Prevents cluttering editor history with complex file paths

### Keyboard Shortcuts
- **Opt+A / Alt+A**: Navigate to previous file type
- **Opt+S / Alt+S**: Navigate to next file type
- **Opt+T / Alt+T**: Open current document in split view
- File type cycling: UI → Component → SCSS → API → Socket

## Project Management

### Project Operations
- **Project Switching**: Quick switch between multiple projects
- **Project Import**: Import `.wizproject` archive files
- **Project Export**: Export projects as `.wizproject` archives
- **Project Deletion**: Safe removal with confirmation dialogs
- **Git Integration**: Clone projects directly from repositories

### Package Management
- **Package Creation**: Create new Portal packages
- **Package Export**: Export packages as `.wizpkg` archives

---

## App Management

### App Creation
- **Source Apps**: Create Page, Component, Layout apps in `src/` directory
- **Portal Apps**: Create apps inside Portal packages with namespace sync
- **Route Creation**: Create routes with flat structure support
- **Create Editors**: Webview-based creation dialogs with form validation

### App Editing
- **Namespace Sync**: Changing namespace automatically renames folder and updates ID
- **Portal Folder Icons**: Unified icon set for Portal package folders
- **App Title Display**: Improved tree item labels showing app titles

---

## Build Integration

- **Auto-Build on Save**: Automatic build triggered when saving files
- **Normal / Clean Build**: Choose build type as needed
- **Build Output Channel**: Real-time build log viewing
- **Python Interpreter**: Configure Python path for build environment

## Architecture

### Module Structure

```
src/
├── core/                   # Core utilities and constants
│   ├── constants.js        # App types, icons, file mappings
│   ├── pathUtils.js        # URI parsing, app folder detection
│   ├── fileUtils.js        # File I/O operations
│   ├── uriFactory.js       # wiz:// URI factory
│   └── webviewTemplates.js # HTML templates for Webviews
├── editor/                 # Webview editors
│   ├── appEditorProvider.js       # Facade for editor management
│   ├── wizFileSystemProvider.js   # wiz:// file system
│   └── editors/                   # Editor implementations
│       ├── editorBase.js          # Abstract base class
│       ├── appEditor.js           # Standard app editor
│       ├── routeEditor.js         # Route editor
│       ├── portalAppEditor.js     # Portal app editor
│       └── portalEditor.js        # Portal package editor
├── explorer/               # Tree view
│   ├── fileExplorerProvider.js    # Main tree data provider
│   ├── models/categoryHandlers.js # Category logic
│   ├── appPatternProcessor.js     # App grouping by type
│   └── wizDragAndDropController.js # Drag & drop
└── extension.js            # Entry point
```

### Design Patterns
- **Facade Pattern**: `AppEditorProvider` centralizes editor management
- **Factory Pattern**: `WizUriFactory` generates virtual URIs
- **Template Method**: `EditorBase` defines lifecycle, subclasses override
- **Strategy Pattern**: Category handlers for Source/Portal/Project
