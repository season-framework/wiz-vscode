# Wiz VSCode Extension v1.3.0 Release Notes

> February 21, 2026

---

## Highlights

**v1.3.0** delivers **MCP native delegation** to VS Code's built-in MCP management, **session-based state management** for multi-user environments, **automatic extension updates** from GitHub releases, and adds **Git clone support for .github directory**.

---

## Breaking Changes

### MCP Server Management → VS Code Native
The extension no longer spawns a separate MCP server process. MCP server lifecycle is now **fully managed by VS Code** through `.vscode/mcp.json`.

- **Enable**: Adds `wiz` server configuration to `mcp.json` — VS Code handles startup automatically
- **Disable**: Removes `wiz` server from `mcp.json` — VS Code handles shutdown
- **State**: Determined by presence of `wiz` entry in `mcp.json`, always in sync with VS Code

---

## New Features

### Automatic Extension Update
The extension checks for the latest version on GitHub at startup and shows an update indicator in the Settings tree.

- **Version Check**: Fetches latest tag from `github.com/season-framework/wiz-vscode/tags`
- **Update Indicator**: `version: v1.2.2 → v1.3.0 ⬆ update` with cloud-download icon
- **One-Click Update**: Downloads vsix from GitHub releases, installs, and prompts reload
- **URL Pattern**: `https://github.com/season-framework/wiz-vscode/releases/download/v{version}/wiz-vscode-{version}.vsix`

### Session-Based State Management
Multiple VS Code instances can now work simultaneously without state conflicts in `.wiz-state.json`.

- **Session ID**: Uses `vscode.env.sessionId` as unique key per VS Code instance
- **State Format**: `{ sessions: { [sessionId]: { workspacePath, currentProject, lastUsed } } }`
- **MCP Server**: Reads the most recently used session's data
- **Auto Cleanup**: Sessions unused for 7+ days are automatically purged
- **Backward Compatible**: Migrates old single-entry format to session-based format

### Git Clone for .github Directory
Right-click the **Copilot** category in the tree to import a Git repository as the `.github` directory.

- **Context Menu**: "Git에서 불러오기" on Copilot category
- **Warning Dialog**: Confirms before replacing existing `.github` directory
- **Auto Cleanup**: Removes `.git` subdirectory after clone (no nested repo history)

### Settings Category (Wiz Settings)
A new **wiz settings** tree category replaces the old version footer, providing centralized access to all settings.

| Item | Description |
|------|-------------|
| `project: {name}` | Current project name (click to copy) |
| `version: v{x.y.z}` | Extension version with update indicator |
| `mcp configuration` | Create or manage MCP settings |
| `clean build` | Clean build with confirmation dialog |
| `python env` | Select Python interpreter for build |
| `python packages` | pip package manager |
| `npm packages` | npm package manager |

## Improvements

### MCP Configuration Management
- **Auto-create**: `mcp.json` is automatically created when enabling MCP server
- **Config Menu**: QuickPick with "설정 보기" and "초기화 하기" options
- **Reset Config**: Deletes and regenerates `mcp.json` with current extension path
- **File Watcher**: Monitors `mcp.json` changes (create/modify/delete) for real-time tree updates

### Build & Project UX
- **Build shortcut**: Build command directly executes normal build (no menu)
- **Clean Build**: Modal warning confirmation before execution
- **Project name**: Real directory case-sensitive name resolution and clipboard copy

## Internal

- Removed `child_process` dependency from `McpManager`
- Removed `outputChannel` and `serverProcess` management
- Added `_hasWizServer()`, `_ensureConfig()`, `_notifyState()` methods
- Version comparison utility `_compareVersions()` in `SettingsCategory`
- GitHub API integration for tag fetching with 5s timeout
- HTTPS download with 302 redirect handling for vsix installation
