# Wiz VSCode Extension v1.2.2 Release Notes

> February 20, 2026

---

## Highlights

**v1.2.2** improves **tree view readability** by displaying app titles instead of IDs for route and portal apps, and adds **MCP dependency management** tools for pip and npm package operations directly through AI agents.

---

## New Features

### Route/Portal App Title Display
Route apps and Portal apps now show their **Title** as the primary label in the file tree, with the ID displayed as a secondary description.

- **Source Route**: `src/route/{id}/` — reads `app.json` title, shows as `Title` `id`
- **Portal Route**: `src/portal/{package}/route/{id}/` — same title display behavior
- **Portal App**: `src/portal/{package}/app/{namespace}/` — reads `app.json` title
- **Fallback**: If no title in `app.json`, displays ID as before
- **Consistent UX**: Matches the existing Standard App behavior in `appGroupItem.js`

### MCP Dependency Management Tools
6 new MCP tools for managing pip and npm packages through AI agents.

**pip Tools**:

| Tool | Description |
|------|-------------|
| `wiz_pip_list` | List installed packages (with `outdated` option for upgradeable packages) |
| `wiz_pip_install` | Install package(s) with optional `upgrade` flag |
| `wiz_pip_uninstall` | Uninstall package(s) |

**npm Tools**:

| Tool | Description |
|------|-------------|
| `wiz_npm_list` | List installed packages (with `outdated` and `global` options) |
| `wiz_npm_install` | Install package(s) with `dev` and `global` options |
| `wiz_npm_uninstall` | Uninstall package(s) with `global` option |

**Smart Environment Detection**:
- **pip**: Auto-detects workspace virtual environment (`venv/`, `.venv/`, `env/`), falls back to system `pip3`
- **npm**: Uses project directory if `package.json` exists, otherwise workspace root; auto-runs `npm init -y` if needed

---

## Improvements

### MCP Server
- **Tool Count**: Expanded from 30 tools to 36 tools
- **Tool Categories**: New "Dependency Management" category added
- **Version specifiers**: Supports `flask>=2.0`, `lodash@4.17.21` etc.

### Tree View
- **Route readability**: Route apps now display human-readable titles instead of cryptic folder names
- **Portal app readability**: Portal apps show title from `app.json` with namespace as description
